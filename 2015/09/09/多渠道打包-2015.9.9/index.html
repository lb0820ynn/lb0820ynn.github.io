Android Studio多渠道打包及自定义打包名称

作为Android开发，在项目上线的时候多数会遇到多渠道包的情况,，因为国内Android市场分发渠道太多，经常一个Apk需要打十几个到几十个渠道包，还好Gradle可以帮我们解决这个问题。

Umeng多渠道打包

以Umeng统计为例，清单文件中会有这样一段

<meta-data
        android:name="UMENG_CHANNEL"
        android:value="${UMENG_CHANNEL_VALUE}"/>

value值就是渠道值，每个渠道包都有一个单独的值对应，所以需要这个值可以根据不同的渠道变化

  • 1.在清单文件umeng渠道value处配置，如下
<meta-data
         android:name="UMENG_CHANNEL"
         android:value="${UMENG_CHANNEL_VALUE}"/>
  • 2.在app下的build.gradle中设置productFlavors
productFlavors {
    GooglePlay {
        manifestPlaceholders = [UMENG_CHANNEL_VALUE: "649"]
    }
    hiapk {
        manifestPlaceholders = [UMENG_CHANNEL_VALUE: "101"]
    }
    shichang91 {
        manifestPlaceholders = [UMENG_CHANNEL_VALUE: "102"]
    }
    gfan {
        manifestPlaceholders = [UMENG_CHANNEL_VALUE: "103"]
    }
    shichang360 {
        manifestPlaceholders = [UMENG_CHANNEL_VALUE: "104"]
    }
    qq {
        manifestPlaceholders = [UMENG_CHANNEL_VALUE: "105"]
    }
    xiaomi {
        manifestPlaceholders = [UMENG_CHANNEL_VALUE: "106"]
    }
    goapk {
        manifestPlaceholders = [UMENG_CHANNEL_VALUE: "107"]
    }
    baidu {
        manifestPlaceholders = [UMENG_CHANNEL_VALUE: "108"]
    }
    appchina {
        manifestPlaceholders = [UMENG_CHANNEL_VALUE: "109"]
    }
}
  • 3.对于渠道值与其Name不一样的采取上面的方式，若一样可以采取下面的简单方式
android {  
    productFlavors {
        xiaomi {}
        _360 {}
        baidu {}
        wandoujia {}
    }  
    productFlavors.all { 
        flavor -> flavor.manifestPlaceholders = [UMENG_CHANNEL_VALUE: name] 
    }
}

  • 4.这个时候在android studio的build中的Generate Signed Apk选项中的Flavors中就可以看到在productFlavors中添加的这些渠道，若全部打出的话则全部选定点击Finish即可

  • 5.同样可以使用Gradle的命令方式来打包在AndroidStudio的命令行里面可以执行gradlew assembleRelease这句命令，(gradlew是gradle wrapper的意思，个人理解为AndroidStudio对Gradle在本地的一层包装)就可以慢慢等apk打包成功了。关于这个命令下面解释一下。

assemble 命令

gradlew assembleDebug 编译并打Debug包
gradlew assembleRelease 编译并打Release的包

除此之外 assemble 还能和 Product Flavor 结合创建新的任务，其实 assemble 是和 Build Variants 一起结合使用的，而 Build Variants = Build Type + Product Flavor ， 举个例子大家就明白了：

如果我们想打包wandoujia渠道的release版本，执行如下命令就好了：

./gradlew assembleWandoujiaRelease
如果我们只打wandoujia渠道版本，则：

./gradlew assembleWandoujia
此命令会生成wandoujia渠道的Release和Debug版本

同理我想打全部Release版本：

./gradlew assembleRelease
这条命令会把Product Flavor下的所有渠道的Release版本都打出来。
打出包的目录在这个位置\app\build\outputs\apk

自定义输出的Apk的包的名称

上线时候遇到一个问题，AndroidStudio打出来的Apk默认都是英文名字，但是上线的工作是运营MM负责，让他们对着英文的apk并准确的上线到各大Android市场，也是难为他们了。但几十个Apk需要自己一个个重命名为以汉字为名的apk，也是很难为我们的，所以gradle的强大之处就显现出来了，看下面代码。

applicationVariants.all { variant ->
    variant.outputs.each { output ->
        def oldFile = output.outputFile
        if (variant.buildType.name.equals('release')) {
            def releaseApkName = ''
            if (variant.productFlavors[0].name.equals('uc')) {
                releaseApkName = 'UC'
            } else if (variant.productFlavors[0].name.equals('GooglePlay')) {
                releaseApkName = '谷歌商店'
            } else if (variant.productFlavors[0].name.equals('hiapk')) {
                releaseApkName = '安卓市场'
            } else if (variant.productFlavors[0].name.equals('shichang91')) {
                releaseApkName = '91助手'
            } else if (variant.productFlavors[0].name.equals('gfan')) {
                releaseApkName = '机锋网'
            } else if (variant.productFlavors[0].name.equals('shichang360')) {
                releaseApkName = '360手机助手'
            } else if (variant.productFlavors[0].name.equals('qq')) {
                releaseApkName = '应用宝(QQ市场)'
            } else if (variant.productFlavors[0].name.equals('xiaomi')) {
                releaseApkName = '小米'
            } else if (variant.productFlavors[0].name.equals('goapk')) {
                releaseApkName = '安智市场'
            } else if (variant.productFlavors[0].name.equals('baidu')) {
                releaseApkName = "百度"
            } else if (variant.productFlavors[0].name.equals('appchina')) {
                releaseApkName = '应用汇市场'
            } else if (variant.productFlavors[0].name.equals('xcar')) {
                releaseApkName = 'M页面'
            } else if (variant.productFlavors[0].name.equals('other')) {
                releaseApkName = '其他'
            } else if (variant.productFlavors[0].name.equals('sogou')) {
                releaseApkName = '搜狗市场'
            } else if (variant.productFlavors[0].name.equals('jinshan')) {
                releaseApkName = '金山手机助手'
            } else if (variant.productFlavors[0].name.equals('leshangdian')) {
                releaseApkName = '乐商店cpt'
            }
            releaseApkName = releaseApkName + ".apk"
            output.outputFile = new File(oldFile.parent, releaseApkName)
        }
    }
}

variant.productFlavors[0].name就是上面所写的productFlavor的Name
这段代码放在app下面的build.gradle文件下android标签里面。当然这还是最简单的应用，不过确实能节省一些时间，通过productFlavor还能实现每个渠道包某些更大方面的不同，包括包名，签名key之类的都可以实现。
